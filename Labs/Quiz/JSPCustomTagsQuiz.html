2.How do custom tags relate to JSTL?
JSTL is a standard library of JSP actions, it includes custom tags. JSP allows developers to create their own actions by custom tags. Component development creates custom functionality that can be packaged and reused by content developers. Customer can extend SimpleTagSupport class and rewrite doTag() method to make a simplest custom tag. Key steps : Define a tag including attributes and body . Write a Tag Library Descriptor (TLD) that the container will read. Write a tag handler class that implements the tag functionality . Use the tag on a JSP page and link it to the tag descriptor
3.What is the role of the URI in the TLD and the taglib directive?
The uri attribute value resolves to a location the container understands. The taglib directive declares that your JSP page uses a set of custom tags, identifies the location of the library, and provides a means for identifying the custom tags in your JSP page.
4.What is a tag handler class?
It is a type of component of tag library. It is a class extends the tag support framework class – javax.servlet.jsp.tagext.TagSupport. To use custom JSP tags, we need to define the tag handler class that defines the tag's behavior.
5.What is the role of attribute setters in a tag handler class?
To store the attribute value in a member variable that is accessible to the other methods of the tag handler.
6. What is the role of the doTag() method in a tag handler class?
DoTag() method is called once and only once for any given tag invocation by the container. All tag logic, iteration and body evaluations occur in this method.
7. What does the operation getJspContext().getOut().write(“Hi Bob”) do when called in a doTag() method?
DoTag() method takes the current JspContext object using the getJspContext() method and uses it to send "Hi Bob!" to the current JspWriter object
8.What does the operation getJspBody().invoke(null) do when called in a doTag() method?
When getJspBody().invoke(null) is called, it is the output resulting from the execution of the tag body's JSP content that gets passed to the client, not the JSP code itself.
